一个纯前端历史策略页面游戏。你将扮演战国七雄之一的君主，在征伐、变法、合纵、休养之间取舍。游戏不追求复杂操作，而是用事件选择题推动历史：每个选项都有收益，也有代价；没有绝对正确，只有现实妥协。

</p>

## 游戏特色

- **七国自由开局**：秦、楚、齐、赵、魏、韩、燕都可以选择，小国也有翻盘机会。
- **不同时代，不同地图**：三家分晋、合纵连横、长平之后的初始疆域不同，国家强弱随时代变化。
- **事件驱动回合**：每回合先处理突发事件，可能是战事、变法、外交、叛乱、灾荒、宫廷危机。
- **主动出兵系统**：不必等事件刷出目标，可以直接选择攻打某个仍存在的国家。
- **五年方针**：国策不再每回合打断节奏，而是约五年一议，决定对外关系、军事方向和国内政策。
- **动态地图**：56 座城池以不规则色块拼接，攻取、失陷、复国都会改变版图。
- **灭国与复国**：国家灭亡后外交界面置灰，若旧党叛乱成功，也可能重新出现在列国之中。
- **攻都大事件**：特定局势下会出现奇袭都城的机会，成功后可能掳君震降，改写战局。
- **君主寿命与继承**：君主会因体质、年龄、国策压力、宫变和宿疾去世；每代君主都有单独评价。
- **谥号史评**：每代君主去世后会根据本代功过定谥，并给出文言风格的史官评价。

## 怎么玩

1. 打开 `index.html`，选择国家、时代和君主气质。
2. 每回合阅读事件，选择一种处理方式。
3. 如果想主动扩张，可在事件卡下方直接选择目标国家出兵。
4. 每隔数年制定一次五年方针，决定联盟、敌国、备战、休养、变法等方向。
5. 看地图版图变化，直到统一、称霸、偏安、亡国，或在某代君主死后自行作结。

## 本地运行

这是纯静态项目，没有构建步骤。

直接双击打开：

```text
index.html
```

或者启动一个本地静态服务器：

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://localhost:4173/
```

## 设计方向

本作的重点不是复杂数值堆叠，而是“君主一念”的历史后果：

- 打仗能扩地，也会伤财、伤民、结怨。
- 休养能稳国，但会放慢扩张。
- 变法能强国，也会激化内部矛盾。
- 结盟能借势，也可能被盟友拖入战火。
- 弱国可能翻盘，强国也可能因过度扩张遭到合纵围攻。
