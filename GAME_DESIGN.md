# 战国策·抉择 — 完整游戏描述文档

## 一、游戏概述

一款纯前端回合制历史策略游戏。玩家扮演战国七雄（秦、齐、楚、赵、魏、韩、燕）的君主，在 6 个可选纪元中开局，通过每回合的选择题驱动历史进程。游戏共 80 回合，约 240 年的统治跨度，支持君主死亡后选嗣继续游玩（继承系统）。

**核心理念：没有标准答案，只有基于现实的妥协。** 每个事件的选择题都应是两难权衡，高风险选项收益更大但后果严重，低风险选项保底但收益平平。

## 二、核心系统

### 2.1 五维国力
- **军事**（0-100）：影响战争胜负、攻城概率
- **经济**（0-100）：影响国力恢复、税收
- **外交**（0-100）：影响结盟成功率、外交关系
- **内政**（0-100）：影响治理效率、法令推行
- **民心**（0-100）：影响叛乱概率、兵源

### 2.2 6 个纪元
| ID | 名称 | 起始年 | 说明 |
|----|------|--------|------|
| 1 | 三家分晋 | 前403 | 七雄格局正式形成 |
| 2 | 围魏救赵 | 前354 | 桂陵、马陵之战 |
| 3 | 合纵连横 | 前318 | 苏秦张仪纵横天下 |
| 4 | 白起拔郢 | 前278 | 秦势如破竹 |
| 5 | 长平决战 | 前260 | 天下命运悬于一线 |
| 6 | 秦灭六国 | 前230 | 天下一统前夕 |

### 2.3 动态风险系统（极其重要）
每个有风险的选项不显示百分比，只用 **单字暗示**：
- `可`（绿色）— 低风险，成功率约 75%+
- `慎`（橙色）— 中风险，成功率约 50-75%
- `险`（红色）— 高风险，成功率 25-50%

**风险计算公式**：基础风险受以下因素实时浮动——
- 军力高 → 风险↓（有兵权能强制执行）
- 民心低 → 风险↑（民怨易反噬）
- 政令高 → 风险↓（朝政清明）
- 经济好 → 风险↓（有资源周转）
- 朝中有不忠之臣 → 每多一人风险+1.5%

**高风险高回报**：风险>0.5 的选项成功时效果放大 1.35 倍；风险>0.25 的放大 1.15 倍。

### 2.4 城池特性与攻城概率
每个城池有 3 个属性：
- **城防**（1-10）：数值越高越难攻破
- **重要性**（1-10）：战略价值
- **类型**：都城、险关、铁城、粮仓、要塞、边城、普通等

**攻城概率公式** = 军力/200 + 城防系数×0.2 + 计策加成/50 + 季节修正
- 有谋士在朝可获计策加成
- 季节影响：冬难夏易
- 概率钳制在 5%-85% 之间

### 2.5 君主健康与继承
- 君主健康值 0-100，每回合自然衰减 1-3（年龄越大越多）
- 军力 >70 时健康衰减 +1（穷兵黩武伤身）
- 健康降至 10 以下触发"王疾大渐"
- 弹出 3 个继承人选项：贤能太子（+内政民心）、勇武公子（+军力）、外戚幼主（+外交）
- 选择后健康恢复 80-100，年龄重置为 20-35，世代数 +1
- 最多传代游玩，总回合上限 80

### 2.6 零值危机
当任一属性降至 0 时，立即触发强制危机事件，覆盖当前回合的普通事件。4 个选项均带高风险：
- 孤注一掷 → 50% 被民变推翻
- 借外力 → 35% 被架空
- 暴力镇压 → 45% 被刺杀
- 下罪己诏 → 最安全但效果有限

### 2.7 外交系统
- 每两国之间有关系值（-100 ~ +100）
- ≥30 友好，≥60 同盟，≤-10 紧张，≤-30 敌对
- 关系影响 AI 攻击倾向
- 地图上：绿线=同盟，红线=敌对

### 2.8 君主死亡（早停策略）
选项中的 rulerFate 字段定义死亡风险：
- 被弑/被毒杀/战死/被掳/被架空/饿死/出使被扣等
- 触发时显示"大限已至"，点击"史笔留此"进入谥号评价
- 国家被灭时立即跳转结局

## 三、事件系统

### 3.1 事件类型
- **历史固定事件**（~50个）：按年份+纪元触发，每件只触发一次
- **随机事件**（~40个）：从池中抽取，用完重置
- **危机事件**：属性归零时强制触发
- **共享事件**：以 `{neighbor}` `{enemy}` `{ally}` 占位符实现动态国名

### 3.2 事件结构（极其重要）
```javascript
{
  id: "unique_id",        // 唯一标识，用于去重
  year: -354,             // 触发年份
  era: [1, 2, 3],         // 适用纪元
  condition: {
    country: "秦",        // 仅该国触发（严格要求！）
    // 或
    countries: ["赵","齐"], // 这些国家触发
    contextCountry: "秦",   // 上下文主角国
    contextMinMilitary: 30, // 主角国军力要求
  },
  title: "围魏救赵",
  text: "半文言描述...",   // 可用 {neighbor}{enemy}{ally} 占位符
  choices: [
    {
      text: "选项A",
      effects: { military: 10, economy: -5 },  // 固定效果
      risk: 0.25,          // 基础风险（会被动态调整）
      chronicle: "史记文字",
      territoryEffect: { gain: true, from: "魏" },  // 攻城
      rulerFate: { type: "战死", chance: 0.2, text: "...", chronicle: "..." }, // 死亡风险
      specialEffect: { chance: 0.15, type: 'character', pool: true }, // 奇遇
    }
  ]
}
```

### 3.3 反模式（严禁出现）
1. **主语视角错误**：事件 `condition` 用 `countries` 包含多个国家时，必须确保选项从所有视角都合理。如果是战争事件，攻击方和防御方要分开写成两个事件，不可混用。
2. **选项有明显偏好**：每个选项都应有得有失，不存在"完美选项"。
3. **秦国事件出现在楚国**：每个事件的 `condition.country` 必须严格匹配。
4. **事件重复出现**：已触发的事件 `id` 存入 `triggeredEvents`，永不复用。
5. **文本模糊**：不得用"邻国""敌国"等模糊词，必须用 `{neighbor}` 等占位符在运行时替换为具体国名。

## 四、地图系统

### 4.1 SVG 地图
- ViewBox 720×560，深色牛皮纸背景
- 仅保留 4 座大山：崤山、太行、泰山、巫山（带名称标注）
- **不要画河流**
- 7 国领土用极淡色块标识（opacity 0.06）
- 每个城池用独立不规则多边形（7-9 边）标识领土范围
- 城池吞并时多边形变色动画

### 4.2 攻击箭头
- 每回合 AI 或玩家发动攻击后，地图上画弯曲箭头
- 金色箭头 = 玩家攻击（带闪烁动画）
- 红色箭头 = AI 攻城成功
- 从 A 国都城指向 B 国被攻击城池

### 4.3 外交连线
- 关系 ≥30 的同盟国之间画绿色虚线箭头
- 关系 ≤-30 的敌对国之间画红色虚线箭头
- 从玩家都城指向对方都城

### 4.4 城池悬停
显示：城名、所属国、类型（都城/险关等）、城防值、重要性

## 五、谥号与评价系统

### 5.1 谥号生成
主谥根据最终国力、疆域、死因综合判定：
- 统一 → 武/桓
- 霸主 → 昭/宣
- 治世 → 文/明/惠
- 守成 → 安/康/定/简
- 亡国 → 愍
- 被弑 → 厉/幽
- 被掳 → 怀
- 病逝 → 勤/景/悼
副谥根据特殊事件附加（灵=变法、襄=拓地、烈=灭国多、思=民心低等）

### 5.2 评价结构（三板斧）
1. **太史公曰**：在位年数 → 军事评价 → 经济评价 → 外交评价 → 君臣关系 → 关键决策 → 论曰总结
2. **后世影响**：用《X 世家》载伪史笔写法，评价其对后世的历史定位
3. **星级**：★-★★★★★

### 5.3 严禁（极其重要）
- **亡国之君不能被评价为"守成之君"**：`!alive` 必须优先级最高，走亡国专属评价。
- 评价文字必须与玩家实际选择挂钩，不能套模板乱出。

## 六、文案风格

- 事件描述：半文言半白话（如：寡人当何以自处？/ 卿以为如何？/ 王怒而疏之）
- 选项文字：精炼文言（10-20 字）
- 史官记述：文言行文（如：王曰「用商鞅变法…」，事遂。军事振，外交损。）
- 太史公评价：仿《史记》体例

## 七、技术架构

### 7.1 文件结构
```
zhan-guo-strategy/
├── index.html          # 单页面入口，三个"屏幕"：开始/游戏/结局
├── css/style.css       # 古风UI，暗色牛皮纸色调
├── js/
│   ├── countries.js    # 七国数据、纪元配置、城池特性、攻城概率公式
│   ├── characters.js   # 50+ 历史人物 + 角色类型图标
│   ├── events.js       # 50+ 历史事件 + 40+ 随机事件 + 特殊人物池
│   ├── state.js        # 全局状态管理
│   ├── engine.js       # 回合引擎、AI、战斗、继承、危机
│   ├── map.js          # SVG 地图渲染、攻击箭头、外交连线
│   ├── ui.js           # 界面渲染、事件展示、结果面板、继承界面
│   ├── epitaph.js      # 谥号算法、太史公评价、后世影响
│   ├── chronicle.js    # 史书回放
│   ├── save.js         # localStorage 存档
│   └── app.js          # 入口逻辑
```

### 7.2 关键规则
- 纯前端，零依赖，单个 index.html 即可运行
- 所有数据用 JSON 结构内嵌在 JS 文件中
- 事件选择 → 结果面板 → 点击继续 → 下一回合的循环
- 存档用 localStorage，JSON 序列化

## 八、数据规模要求
- 历史事件：≥50 个，覆盖 6 个纪元，每个纪元每国家 ≥5 个
- 随机事件：≥40 个，涵盖战争/外交/内政/人才/奇遇
- 历史人物：≥50 个，按真实年份出生死亡登场
- 特殊奇遇人物：7 国各 ≥4 个，随机事件概率触发
- 城池：≥80 个，每个有独立特性和多边形
